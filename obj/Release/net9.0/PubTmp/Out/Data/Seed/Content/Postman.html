Postman — Beginner Tutorial

1. What is Postman?

Postman is a powerful API development environment that helps you build, test, and document APIs. It provides an easy UI to send HTTP requests, inspect responses, write tests, and automate workflows.

2. Installing Postman

Download Postman Desktop or use the web app:

https://www.postman.com/downloads/

Alternatively, install via package managers (macOS Homebrew example):

brew install --cask postman

3. Postman Interface

  • Sidebar: Collections, APIs, Environments, History
  • Request Builder: Method, URL, Params, Authorization, Headers, Body
  • Response Panel: Status, Headers, Body, Cookies, Timings
  • Console: Debug requests and scripts (View → Show Postman Console)

4. Making Requests

Basic GET request:

                    GET https://jsonplaceholder.typicode.com/posts/1

POST request (JSON body):

                    POST https://jsonplaceholder.typicode.com/posts
                    Headers:
                      Content-Type: application/json

                    Body (raw - JSON):
                    {
                      "title": "foo",
                      "body": "bar",
                      "userId": 1
                    }

Use Params tab for query strings, Headers tab for custom headers, and Body tab for payloads.

5. Authentication

Postman supports many auth methods in the Authorization tab:

  • API Key: Add in header or query param
  • Bearer Token: Set Authorization → Bearer Token
  • Basic Auth: Username & password
  • OAuth 2.0: Use Postman's OAuth helpers to obtain tokens
                    // Example: set Authorization header manually
                    Authorization: Bearer <your-token>

6. Collections & Folders

Collections group related requests. Good for organizing endpoints and sharing with teammates.

  • Create a new collection → Add requests to it
  • Use folders inside a collection for structure
  • Export/import collections (JSON) for sharing

7. Environments & Variables

Environments store variables (like base URLs, tokens) for different contexts (dev/staging/prod).

                    // Example variables
                    baseUrl = https://api.example.com
                    apiKey = 123abc

                    // Use in requests:
                    {{baseUrl}}/users
                    Header: x-api-key: {{apiKey}}

Scopes: Global, Environment, Collection, Local, Data

8. Tests & Scripts

Postman uses JavaScript for pre-request scripts and tests. Tests run after the response and can assert status, response body, headers, etc.

                    // Example tests
                    pm.test("Status is 200", function () {
                      pm.response.to.have.status(200);
                    });

                    pm.test("Body has userId", function () {
                      var json = pm.response.json();
                      pm.expect(json.userId).to.eql(1);
                    });

Pre-request scripts can set or modify variables before a request runs:

                    // Pre-request example: set timestamp
                    pm.environment.set("ts", Date.now());

9. Workflows & Automation

Postman can automate tests and workflows using:

  • Collection Runner — run a collection with data files (CSV/JSON)
  • Monitors — scheduled runs on the cloud
  • Newman — CLI runner for CI integration
                    // Run collection with Newman
                    newman run mycollection.json -e devEnvironment.json
