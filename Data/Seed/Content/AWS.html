AWS — Beginner to Intermediate Tutorial

1. Introduction to AWS

Amazon Web Services (AWS) is a cloud platform offering compute, storage, database, networking, analytics, machine learning, and many other services on-demand.

  • Launch resources in minutes, pay-as-you-go model.
  • Global infrastructure: regions and availability zones (AZs).
  • Used for web hosting, data processing, DevOps, ML, and more.

2. Core Services

  • EC2: Virtual machines (instances) with many instance types (CPU, memory, GPU).
  • S3: Object storage — durable, scalable, widely used for static assets and backups.
  • RDS: Managed relational databases (MySQL, PostgreSQL, MariaDB, Oracle, SQL Server).
  • VPC: Virtual Private Cloud — isolated network in AWS.
  • IAM: Identity and Access Management for users, roles, and policies.
  • Lambda: Serverless functions — pay per execution.

3. Identity & Access Management (IAM)

IAM controls who can do what in your AWS account.

  • Use least privilege: grant only needed permissions.
  • Prefer roles for applications and cross-account access.
  • Enable MFA for root and privileged users.
  • Use IAM policies (JSON) and groups for organization.

4. VPC & Networking

VPC provides network isolation and control:

  • Subnet types: public (with Internet Gateway), private (no direct Internet access).
  • Use NAT Gateway/Instances for outbound access from private subnets.
  • Security Groups (stateful) and Network ACLs (stateless) control traffic.
  • Peering & Transit Gateway for connecting VPCs and accounts.

5. Compute Options (EC2, ECS, Lambda)

  • EC2: Full control VMs — choose AMIs, instance types, EBS volumes.
  • ECS / EKS: Container orchestration — ECS (Fargate or EC2), EKS (Kubernetes).
  • Lambda: Serverless for event-driven workloads; supports many runtimes.

Choose based on control needs, scaling, and operational overhead.

6. Storage Services (S3, EBS, EFS)

  • S3: Object store with versioning, lifecycle policies, and cross-region replication.
  • EBS: Block storage for EC2 (persistent volumes). Snapshots for backups.
  • EFS: Managed NFS for shared file storage across instances.

7. Databases (RDS, DynamoDB)

  • RDS: Managed SQL databases — automated backups, Multi-AZ for HA.
  • DynamoDB: Serverless NoSQL with single-digit ms latency at scale.
  • Aurora: MySQL/Postgres-compatible, high-performance managed DB.

8. Serverless & Containers

  • Serverless benefits: no servers to manage, automatic scaling, cost efficiency for intermittent workloads.
  • Containers benefits: predictable runtimes, portability, good for microservices.
  • Use event sources (S3, API Gateway, SQS) to trigger Lambda functions.

9. Deployment & CI/CD

  • CodeDeploy / Elastic Beanstalk: Managed deployment options for servers and apps.
  • CodePipeline: CI/CD orchestration integrating CodeBuild and CodeDeploy or third-party tools (GitHub Actions, Jenkins).
  • Infrastructure as Code: CloudFormation / AWS CDK / Terraform for reproducible infra.

10. Monitoring & Logging

  • CloudWatch: Metrics, logs, alarms, dashboards.
  • CloudTrail: API activity logging for auditing and compliance.
  • Centralize logs (CloudWatch Logs, Kinesis, or third-party SIEM) and set meaningful alerts.

11. Security Best Practices

  • Use MFA and avoid long-term access keys; prefer temporary STS credentials.
  • Encrypt data at rest (KMS) and in transit (TLS).
  • Use GuardDuty, Inspector, and AWS Config for threat detection and compliance checks.
  • Segment workloads across accounts using AWS Organizations and SCPs.
  • Harden AMIs and limit surface area (few open ports, security group least privilege).

12. Cost & Billing

  • Monitor usage with AWS Cost Explorer and Budgets.
  • Choose right-sizing (instance types, reserved instances, savings plans) for long-running workloads.
  • Use lifecycle policies (S3) and tiering to optimize storage costs.

13. Hands-on: Getting Started

  1. Create an AWS account (use billing alerts) and enable MFA.
  2. Install & configure AWS CLI: aws configure.
  3. Launch a small EC2 instance (t2.micro / t3.micro) in a test VPC; SSH using key pair.
  4. Create an S3 bucket and upload a static website object (or host a static site using S3 + CloudFront).
  5. Try a serverless function: create a simple Lambda triggered by S3 upload or API Gateway.
  6. Explore CloudFormation or CDK: create a simple stack to automate the above.
