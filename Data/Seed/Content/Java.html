Java — Beginner to Intermediate Tutorial

1. Introduction

Java is a class-based, object-oriented programming language designed for portability and reliability.

  • Developed by Sun Microsystems (1995).
  • “Write Once, Run Anywhere” — Java runs on the JVM.
  • Used in Android apps, backend systems, and enterprise solutions.

2. Setup & Installation

                    # Download Java JDK:
                    https://www.oracle.com/java/technologies/javase-downloads.html

                    # Check version:
                    java -version
                    javac -version

Use IDEs like IntelliJ IDEA, Eclipse, or VS Code for productivity.

3. Basic Syntax

                    public class HelloWorld {
                        public static void main(String[] args) {
                            System.out.println("Hello, Java!");
                        }
                    }
  • All code must be inside a class.
  • main() is the program entry point.
  • Statements end with a semicolon.

4. Variables & Data Types

                    int age = 25;

Java is statically typed — types must be declared explicitly.

6. Control Flow

                    if (x > 0) {
                        System.out.println("Positive");
                    } else if (x == 0) {
                        System.out.println("Zero");
                    } else {
                        System.out.println("Negative");
                    }

                    for (int i = 0; i < 5; i++) {
                        System.out.println(i);
                    }

                    while (x > 0) {
                        x--;
                    }

7. Object-Oriented Programming

                    class Animal {
                        String name;
                        Animal(String name) { this.name = name; }

                        void speak() {
                            System.out.println(name + " makes a sound");
                        }
                    }

                    class Dog extends Animal {
                        Dog(String name) { super(name); }

                        @Override
                        void speak() {
                            System.out.println(name + " barks");
                        }
                    }

                    public class Main {
                        public static void main(String[] args) {
                            Dog d = new Dog("Rex");
                            d.speak();
                        }
                    }

Java supports encapsulation, inheritance, and polymorphism.

8. Collections Framework

                    import java.util.*;

                    List names = new ArrayList<>();
                    names.add("Alice");
                    names.add("Bob");

                    for (String n : names)
                        System.out.println(n);

                    Map ages = new HashMap<>();
                    ages.put("Alice", 25);
                    ages.put("Bob", 30);
                    System.out.println(ages.get("Alice"));

The Collections Framework provides dynamic data structures like lists, sets, and maps.

9. Exceptions

                    try {
                        int result = 10 / 0;
                    } catch (ArithmeticException e) {
                        System.out.println("Error: " + e.getMessage());
                    } finally {
                        System.out.println("Cleanup complete");
                    }

Always catch specific exceptions and use finally for cleanup operations.

10. File Handling

                    import java.io.*;

                    try {
                        FileWriter writer = new FileWriter("output.txt");
                        writer.write("Hello File!");
                        writer.close();

                        BufferedReader reader = new BufferedReader(new FileReader("output.txt"));
                        System.out.println(reader.readLine());
                        reader.close();
                    } catch (IOException e) {
                        e.printStackTrace();
                    }

11. Best Practices

  • Use meaningful class and variable names.
  • Follow standard naming conventions (camelCase, PascalCase).
  • Handle exceptions gracefully.
  • Prefer interfaces and abstract classes for flexibility.
  • Document your code with Javadoc comments.
